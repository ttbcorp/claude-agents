# TTB Advisor

Sonnet으로 진행하다 복잡한 결정이나 막힌 상황에서 Opus 리뷰어에 에스컬레이션한다.
토큰 비용 절감을 위해 `/compact` 실행을 사용자에게 먼저 요청하고, 확인 후 `advisor()`를 호출한다.

## 워크플로우

### Phase 0: 컨텍스트 요약 출력

사용자에게 `/compact` 요청 전, 현재 작업 상황을 아래 표 형식으로 먼저 정리하여 보여준다.
사용자가 `/compact` 실행 후 무슨 맥락이 압축되는지 확인할 수 있도록 한다.

| 항목 | 내용 |
|------|------|
| 작업 목표 | 현재 진행 중인 작업의 핵심 목표 한 줄 요약 |
| 진행 상태 | 어디까지 완료됐는지 |
| 막힌 지점 / 결정 사항 | advisor에게 물어볼 핵심 질문 또는 결정 포인트 |

### Phase 1: /compact 요청

아래 메시지를 사용자에게 출력한다:

```
advisor() 호출 전 토큰 절감을 위해 /compact를 먼저 실행해주세요.

  /compact

실행 후 "완료" 또는 "했어" 라고 입력하면 Opus 리뷰를 시작합니다.
```

사용자의 응답을 기다린다.

### Phase 2: 확인 판단

사용자 응답에서 아래 신호를 감지한다:

| 응답 유형 | 신호 예시 | 처리 |
|---------|---------|------|
| 완료 확인 | "완료", "했어", "ok", "done", "ㅇㅇ", "응" | Phase 3으로 진행 |
| 건너뛰기 | "그냥 해줘", "skip", "바로 해줘" | Phase 3으로 진행 (토큰 주의 안내 후) |
| 취소 | "취소", "안 할게", "됐어" | 스킬 종료 |

### Phase 3: advisor() 호출

```python
advisor()
```

advisor 결과를 받아 사용자에게 보고한다.

### Phase 4: 결과 보고

advisor 피드백을 아래 형식으로 정리한다:

| 항목 | 내용 |
|------|------|
| 핵심 권고 | advisor가 제시한 가장 중요한 방향 |
| 주의 사항 | 놓친 부분, 리스크, 재고해야 할 점 |
| 다음 행동 | 권고에 따라 바로 실행할 수 있는 구체적 액션 |

advisor 결과와 내 관찰이 충돌할 경우, 충돌 내용을 명시하고 추가 advisor 호출로 해소한다.

## 에러 핸들링

| 상황 | 처리 |
|------|------|
| 사용자가 응답 없이 다른 요청 | 현재 Phase 1에서 대기 중임을 안내하고 재확인 |
| advisor() 결과가 없거나 불명확 | "advisor 응답을 받지 못했습니다. 다시 시도할까요?" 안내 |
